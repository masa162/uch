【改訂版】メディア機能 開発タスクリスト
1. 【基盤構築】インフラストラクチャの準備 (Task-000)
目的: 開発を開始するための前提条件を整える。

作業内容:

[ ] Cloudflare R2:

uchinokiroku-assetsバケットを作成する。

Object Read & Write権限でAPIトークンを作成し、キーを安全に保管する。

[ ] Next.js:

.env.localファイルに、R2のバケット名、アカウントID、APIキーなどの環境変数を設定する。

[ ] PostgreSQL:

メディア情報を管理するためのmediaテーブルとmedia_optimizedテーブルを設計し、作成する。（設計書参照）

2. 【コア機能】メディアアップロード機能の実装 (Task-001)
目的: 家族がブラウザから写真や動画をアップロードできる、プロジェクトの核となる機能を提供する。

作業内容:

[ ] UI作成:

ファイルを選択できるシンプルなアップロードUIコンポーネントを作成する (ImageUploader.js)。

[ ] API開発 (署名付きURL発行):

Next.jsのAPIルート (/api/media/generate-upload-url) を作成する。

このAPIは、ファイル名と種類を受け取り、Cloudflare R2へのアップロードを許可する署名付きURLを生成して返す。

[ ] クライアント側ロジック:

UIから上記APIを呼び出して署名付きURLを取得する。

取得したURLを使い、ファイルをブラウザからR2へ直接アップロードする。

[ ] API開発 (DB記録):

アップロード完了後、ファイル情報（ユニークキー、元のファイル名など）をDBに保存するためのAPIルート (/api/media/upload-complete) を作成し、クライアントから呼び出す。

3. 【バックエンド】AI執事による自動最適化 (Task-002)
目的: アップロードされたメディアを管理・閲覧に適した形式に自動変換し、管理コストをゼロにする。

作業内容:

[ ] AI執事スクリプト開発:

VPS上で動作する独立したNode.jsスクリプトを作成する。

スクリプトは、DBをチェックしてstatusがpendingの未処理ファイルを見つける。

R2のoriginals/フォルダからファイルをダウンロードする。

画像: sharpライブラリを使い、リサイズとWebP形式への変換を行う。

動画: ffmpegを使い、MP4形式への変換と圧縮を行う。

処理後のファイルをR2のoptimized/フォルダにアップロードする。

DBのstatusをoptimizedに更新し、最適化済みファイルの情報を記録する。

[ ] 自動実行設定:

VPSのcronを使い、このAI執事スクリプトを定期的（例: 5分ごと）に自動実行するよう設定する。

4. 【UI/UX】メディアギャラリーの作成 (Task-003)
目的: アップロードされ、最適化された写真や動画を快適に閲覧できるインターフェースを提供する。

作業内容:

[ ] API開発 (一覧取得):

DBから最適化済みのメディア情報一覧を取得するためのAPIルート (/api/media) を作成する。

[ ] UI作成:

取得したAPIデータを元に、写真や動画を一覧表示するギャラリーページを作成する。

画像や動画のURLは、R2の公開URL（または署名付き閲覧URL）を動的に生成して表示する。

[ ] 機能実装:

まずは日付の新しい順に表示する。将来的には、日付やタグによる絞り込み・検索機能を実装する。

5. 【拡張機能】AI執事による自動タグ付け (Task-004)
目的: AIを活用して写真の内容に基づいたタグを自動付与し、後の検索性や思い出の発見を容易にする。

作業内容:

[ ] AI執事スクリプト拡張:

Task-002のスクリプトを拡張する。

メディアの最適化処理が完了した後、その画像を画像認識AI（Dify経由のLLMやGoogle Cloud Vision APIなど）に送信する。

[ ] DB連携:

AIから返ってきたタグ情報（例: 「海」「犬」「笑顔」）をDBに保存し、対象のメディアと紐付ける。

[ ] UI/UX拡張:

Task-003のギャラリーページに、タグによる絞り込み機能を追加する。

# TSK-070: [本番UX改善] 認証画面のちらつきを調査・修正

## 概要
本番環境で認証画面にちらつきが発生する問題の調査と修正を実施。

## 実施内容

### 1. 問題調査
- **認証フロー確認**: AuthenticatedLayout、AuthContext、Providers の実装を調査
- **ちらつき原因特定**: セッション取得後のリダイレクト処理で一瞬の空白画面が発生

### 2. 根本原因特定
**問題箇所**: `src/components/AuthenticatedLayout.tsx:27-41`

**処理フロー**:
1. `status === 'loading'` → ローディング画面表示
2. `!session` → `null` を返す（空白画面）← **ちらつきの原因**
3. useEffect でリダイレクト処理
4. サインインページへ遷移

**問題**: ステップ2で `null` を返すため、一瞬空白画面が表示される

### 3. 修正実装

```typescript
// 修正前
if (status === 'loading') {
  return <LoadingScreen />
}
if (!session) {
  return null  // ← 空白画面の原因
}

// 修正後
if (status === 'loading' || !session) {
  return <LoadingScreen message={
    status === 'loading' ? '読み込み中...' : 'サインイン画面に移動中...'
  } />
}
```

### 4. 改善内容
- **一貫性**: 認証フロー全体で統一されたローディング画面を表示
- **UX向上**: 空白画面の除去により、スムーズな画面遷移を実現
- **明確性**: 状態に応じた適切なメッセージ表示

- ✅ ちらつき問題解消

# TSK-047: 本番環境での認証スキップ設定の確認と修正

## 1. 目的
本番環境で意図せず認証がスキップされている問題を調査し、`/articles/new` ページで「ログインが必要です」と表示されるバグを修正する。

## 2. 詳細
ユーザーからの報告によると、本番環境（`uchinokiroku.com`）にアクセスしても、あいことばや認証が求められない。これはTSK-035「ローカル環境での認証完全除外」でローカル開発環境向けに認証をスキップするように設定した影響が本番環境にも及んでいる可能性がある。

また、認証がスキップされているにも関わらず、`/articles/new` ページでは「ログインが必要です」と表示され、記事投稿機能が利用できない。これは認証スキップの意図と異なる挙動であり、バグとして修正が必要。

## 3. 調査・修正項目
- 本番環境のNext.jsアプリケーションが、`NEXT_PUBLIC_SKIP_AUTH` などの環境変数をどのように解釈しているか確認する。
- `src/lib/auth.ts` や `src/components/PasswordGate.tsx` など、認証関連のコードが本番環境でどのように動作しているか調査する。
- `/articles/new` ページで「ログインが必要です」と表示される原因を特定し、認証スキップが有効な場合は記事投稿ページにアクセスできるように修正する。
- もしくは、本番環境では認証スキップを無効にし、あいことば/認証が正しく機能するように設定を戻す（ただし、これはユーザーの「家族ローンチ直前で再度、あいことば、認証設定をする」という指示と矛盾しないように注意する）。

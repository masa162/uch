# タスク詳細: TSK-062

**ID**: `TSK-062`
**タイトル**: 【Frontend】書き込み機能の利用制限UIの実装
**ステータス**: 未着手
**優先度**: 高
**担当者**: Claude Code
**関連タスク**: TSK-060, TSK-061

## 1. タスクの目的

ゲストとしてログインしているユーザーに対し、UI上で「どの機能が利用できて、どの機能が利用できないか」を明確に伝える。利用できない機能をクリックした際には、APIエラーをただ待つのではなく、フロントエンド側で即座にログイン/新規登録を促すメッセージを表示し、スムーズなユーザー体験を提供する。

## 2. 背景

`TSK-060`でAPIレベルのアクセス制限は実装されたが、UI上ではゲストも正規ユーザーも同じようにボタンが押せてしまう。ゲストがボタンを押した結果、単にAPIエラーが返ってくるだけでは不親切である。ユーザーが「なぜこの機能が使えないのか」を即座に理解し、次のアクション（登録）に移れるように導線を設計する必要がある。

## 3. 具体的な作業内容

### 3.1. 共通の権限チェック＆モーダル表示関数の作成

アプリケーション全体で再利用可能な、権限をチェックして必要であればログインモーダルを表示する、という共通の関数またはカスタムフックを作成する。

```typescript
// 例: src/hooks/useAuthAction.ts

import { useSession } from 'next-auth/react';
import { useRouter } from 'next/navigation';

export const useAuthAction = () => {
  const { data: session } = useSession();
  const router = useRouter();

  const runAuthAction = (action: () => void) => {
    if (session?.user.role === 'GUEST' || !session) {
      // ここでdaisyUIのモーダルを表示するなどの処理
      // alertは仮実装
      if (confirm('この機能の利用にはユーザー登録が必要です。ログインページに移動しますか？')) {
        router.push('/auth/signin');
      }
    } else {
      action();
    }
  };

  return { runAuthAction };
};
```

### 3.2. 各書き込み系コンポーネントへの適用

作成した`useAuthAction`フックを、書き込み操作を行うすべてのコンポーネントに適用する。

*   **対象コンポーネントの例**:
    *   いいねボタン
    *   コメント投稿フォーム
    *   記事の新規作成ボタン
    *   記事の編集・削除ボタン

*   **実装例（いいねボタン）**:

    ```tsx
    // ... いいねボタンコンポーネント内
    import { useAuthAction } from '@/hooks/useAuthAction';

    const { runAuthAction } = useAuthAction();

    const handleLike = () => {
      // 実際のいいねAPIを叩く処理
      console.log('いいね処理を実行！');
    };

    return (
      <button onClick={() => runAuthAction(handleLike)}>
        いいね！
      </button>
    );
    ```

## 4. 完了の定義

*   ゲストログイン状態で、記事投稿、コメント、いいね等のボタンをクリックすると、APIリクエストが送信されることなく、ログイン/新規登録を促すメッセージ（モーダルが望ましい）が表示されること。
*   正規ユーザーとしてログインしている場合は、通常通り機能が実行されること。

## 5. 検証方法

PMがローカル環境でアプリケーションを操作し、以下の2つのシナリオで動作を確認する。
1.  **ゲストとしてログイン**: 各書き込みボタンをクリックし、ログインを促すモーダルが表示されることを確認する。
2.  **正規ユーザーとしてログイン**: 各書き込みボタンをクリックし、機能が正常に動作することを確認する。

# TSK-034: レガシー資産のクリーンアップと棚卸し

## 概要
本タスクは、リポジトリ移行（uchiからuch）に伴い発生した、各環境における不要なファイル、設定、リソースの特定とクリーンアップを目的とします。これにより、プロジェクトの健全性を維持し、将来的なメンテナンスコストを削減します。

## 目的
- 不要なファイルの削除
- 未使用の設定の整理
- リソースの最適化
- 環境の一貫性と再現性の向上

## チェックアップ項目

### 1. ローカル開発環境
- **Git履歴の整理**:
    - 不要なブランチの削除 (`git branch -d <branch_name>`)
    - 古いコミット履歴の確認と、必要に応じたsquash/rebaseの検討
- **一時ファイル・キャッシュ**:
    - `node_modules` 以外のキャッシュディレクトリ (`.next/cache`, `dist` など) の確認とクリーンアップ
    - ログファイル (`*.log`) の確認と整理
- **未使用の依存関係**:
    - `package.json` に記載されているが、実際には使用されていないパッケージの特定と削除 (`npm prune` や `depcheck` などのツール活用を検討)
- **古い設定ファイル**:
    - `.env.local` やその他の設定ファイル内の未使用変数、コメントアウトされた古い設定の確認と整理
- **テストデータ・サンプルファイル**:
    - 開発中に作成された一時的なテストデータやサンプルファイルの確認と削除

### 2. Docker環境
- **未使用のイメージ・コンテナ**:
    - `docker images -a` で未使用イメージの確認と削除 (`docker rmi <image_id>`)
    - `docker ps -a` で停止中のコンテナの確認と削除 (`docker rm <container_id>`)
- **古いボリューム・ネットワーク**:
    - `docker volume ls` で未使用ボリュームの確認と削除 (`docker volume rm <volume_name>`)
    - `docker network ls` で未使用ネットワークの確認と削除 (`docker network rm <network_name>`)
- **`Dockerfile` / `docker-compose.yml` の最適化**:
    - 不要なビルドステップ、未使用の環境変数、ポートマッピングの確認と削除
    - マルチステージビルドの導入など、イメージサイズの最適化検討
- **ビルドキャッシュのクリーンアップ**:
    - `docker builder prune` の実行

### 3. VPS (本番/ステージング) 環境
- **古いデプロイディレクトリ・ビルドアーティファクト**:
    - 過去のデプロイで残された古いアプリケーションディレクトリやビルド成果物の確認と削除
- **未使用のユーザー・グループ**:
    - システム上に存在するが、現在使用されていないユーザーアカウントやグループの確認と削除
- **ログファイル・バックアップファイル**:
    - 古いログファイルや不要なバックアップファイルの確認と削除
- **不要なcronジョブ**:
    - 過去のタスクで設定されたが、現在不要なcronジョブの確認と削除
- **ファイアウォールルール**:
    - `ufw status` などで、現在使用されていないが開放されているポートの確認と閉鎖

### 4. ポート管理
- **未使用ポートの特定**:
    - ローカル、Docker、VPSの各環境で、現在アプリケーションで使用されていないが、何らかの理由で開放されているポートの特定
    - `netstat -tulnp` や `lsof -i :<port>` などのコマンド活用
- **ポート競合の解消**:
    - 複数のサービスが同じポートを使用しようとしていないか確認し、必要に応じてポート番号の変更を検討

## 実施手順
1.  各環境のチェックアップ項目を確認し、現状を把握する。
2.  不要と判断されたファイル、設定、リソースを特定する。
3.  削除や変更を行う前に、必ずバックアップを取得する。
4.  変更を適用し、アプリケーションの動作に影響がないことを確認する。

## 完了基準
- 各環境のチェックアップ項目が網羅的に確認され、不要な資産が特定・整理されたこと。
- プロジェクトの健全性が向上し、不要なリソースが削減されたこと。

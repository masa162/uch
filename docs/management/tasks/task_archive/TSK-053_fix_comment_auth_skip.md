# TSK-053: 記事コメント投稿時の認証スキップ対応

## 1. 目的
開発環境で記事コメント投稿時に「ログインが必要です」と表示される問題を修正する。

## 2. 詳細
ユーザーからのフィードバックによると、ローカル環境（開発環境）で記事ページにてコメントを入力し「コメントする」ボタンをクリックすると、「ログインが必要です」というメッセージが表示される。これは、開発効率を向上させるために設定されている認証スキップが、コメント投稿APIに適用されていないためと考えられる。

TSK-047「本番環境での認証スキップ設定の確認と修正」やTSK-051「記事編集機能の動作確認と改善」でAPIへの認証スキップ対応を行ったのと同様に、コメント投稿APIにも開発環境での認証スキップロジックを適用する必要がある。

## 3. 調査・修正項目
- コメント投稿API（`/api/articles/[slug]/comments/route.ts` など）の認証ロジックを確認する。
- 開発環境 (`process.env.NODE_ENV === 'development' && process.env.NEXT_PUBLIC_SKIP_AUTH === 'true'`) の場合に、ダミーユーザーで認証をスキップするロジックを追加する。
- コメント投稿が正常に完了することを確認する。

- `src/app/api/articles/[slug]/comments/route.ts` (コメント投稿API)

# TSK-045: 本番環境のOAuth設定

## 1. タスク概要

TSK-043でデプロイされた本番環境において、OAuth（Google, LINE）認証が機能していない。原因は、環境変数に必要なクライアントIDやシークレットが設定されていないためである。
本タスクでは、本番用の認証情報を設定し、ソーシャルログイン機能を有効化することを目的とする。

## 2. 背景

- **関連タスク**: TSK-043_vps_initial_deployment.md
- **問題**: デプロイ時のログに `The "GOOGLE_CLIENT_ID" variable is not set.` といった警告が多数表示されている。
- **現状**: VPS上の `.env.production` ファイル内のOAuth関連の変数が、初期値のプレースホルダーのままである。

## 3. 作業内容

1.  **認証情報の安全な取得**:
    -   プロジェクトオーナーに連絡を取り、本番環境用の以下の認証情報を安全な方法で受け取る。
        -   `GOOGLE_CLIENT_ID`
        -   `GOOGLE_CLIENT_SECRET`
        -   `LINE_CHANNEL_ID`
        -   `LINE_CHANNEL_SECRET`
    -   **注意**: これらの情報は機密情報であるため、チャットログやバージョン管理下に直接保存してはならない。

2.  **本番環境変数の設定**:
    -   VPSにSSHで接続する。
    -   デプロイされたプロジェクトディレクトリ (`/root/uch`) に移動する。
    -   `vi` や `nano` などのエディタで `.env.production` ファイルを開く。
        ```bash
        vi /root/uch/.env.production
        ```
    -   手順1で取得した認証情報を、対応する環境変数に設定し、保存する。

3.  **アプリケーションの再起動**:
    -   変更した環境変数をアプリケーションに反映させるため、Dockerコンテナを再起動する。
        ```bash
        cd /root/uch
        docker compose restart app
        ```

4.  **動作確認**:
    -   （TSK-044完了後）本番URLにブラウザでアクセスする。
    -   ログインページを開き、「Googleでログイン」「LINEでログイン」ボタンが表示されることを確認する。
    -   各ボタンをクリックし、それぞれの認証フローが正常に完了し、アプリケーションにログインできることを確認する。

## 4. 完了条件

-   本番環境のアプリケーションで、Google OAuth認証が正常に完了すること。
-   本番環境のアプリケーションで、LINEログイン認証が正常に完了すること。

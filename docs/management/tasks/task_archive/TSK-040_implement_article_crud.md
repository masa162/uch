# 作業指示書: TSK-040

**件名:** 記事管理(CRUD)機能の基本実装

## 1. タスク概要

- **タスクID:** TSK-040
- **担当者:** Claude Code
- **優先度:** 最高
- **関連ドキュメント:**
    - `docs/仕様書/レイアウト関連仕様書.md`
    - `docs/仕様書/設計思想.md`

## 2. 目的

アプリケーションの中核機能である、記事の作成 (Create)、一覧表示 (Read)、更新 (Update)、削除 (Delete) の一連のプロセスを実装する。これにより、ユーザーは自身の思い出を記録し、管理できるようになる。

## 3. 開発フェーズ

本タスクは、バックエンドの準備からフロントエンドの実装まで、以下の4つのフェーズで段階的に進める。

---

### **Phase 1: APIの整備 (バックエンド)**

まず、記事の取得（単一）、更新、削除を行うためのAPIエンドポイントを準備する。

- **作業ファイル:** `src/app/api/articles/[slug]/route.ts` を新規作成する。

#### 1.1. 単一記事の取得 (GET)
- **エンドポイント:** `GET /api/articles/[slug]`
- **処理:**
    1.  URLから `slug` を受け取る。
    2.  `prisma.article.findUnique` を使用し、`slug` に一致する記事をデータベースから取得する。
    3.  見つかった記事データをJSONで返す。
    4.  記事が見つからない場合は404エラーを返す。

#### 1.2. 記事の更新 (PUT)
- **エンドポイント:** `PUT /api/articles/[slug]`
- **処理:**
    1.  `getServerSession` で認証状態を確認し、非ログインユーザーは401エラーを返す。
    2.  URLから `slug` を、リクエストボディから更新データ（`title`, `content`など）を受け取る。
    3.  対象記事の所有者と、現在のセッションユーザーが一致するか確認する。一致しない場合は403エラー（権限なし）を返す。
    4.  `prisma.article.update` を使用し、記事を更新する。
    5.  更新後の記事データをJSONで返す。

#### 1.3. 記事の削除 (DELETE)
- **エンドポイント:** `DELETE /api/articles/[slug]`
- **処理:**
    1.  認証状態と記事所有者を確認する（PUTと同様）。
    2.  `prisma.article.delete` を使用し、記事を削除する。
    3.  成功メッセージ（例: `{ message: '削除しました' }`）と共に200ステータスを返す。

---

### **Phase 2: 記事一覧ページの実装 (フロントエンド)**

- **作業ファイル:** `src/app/articles/page.tsx` を新規作成する。

#### 2.1. データ取得
- `fetch` を使用して `/api/articles` (GET) を呼び出し、記事の一覧データを取得する。

#### 2.2. レイアウト
- `レイアウト関連仕様書.md` の「記事一覧ページ」の仕様に基づき、グリッドレイアウトで記事カードを並べる。
- 各カードには、タイトル、概要、投稿日などを表示する。

#### 2.3. リンク設定
- 各記事カード全体を、記事詳細ページ (`/articles/[slug]`) へのリンクとする。
- ダッシュボード (`/`) の「記事一覧」カードから、この `/articles` ページへリンクする。

---

### **Phase 3: 新規作成ページの実装 (フロントエンド)**

- **作業ファイル:** `src/app/articles/new/page.tsx` を修正・実装する。

#### 3.1. フォームの構築
- `レイアウト関連仕様書.md` の「フォームページ」の仕様に基づき、UIを構築する。
- フォームには、タイトル、内容（Markdown対応）、タグなどの入力フィールドを設ける。

#### 3.2. データ送信
- フォームの送信ボタンがクリックされたら、`/api/articles` (POST) にデータを送信する。
- 投稿が成功したら、レスポンスとして返された新しい記事の `slug` を元に、記事詳細ページ (`/articles/[slug]`) へリダイレクトする。

#### 3.3. リンク設定
- ダッシュボード (`/`) の「新しい記事」カードから、この `/articles/new` ページへリンクする。

---

### **Phase 4: 詳細・編集・削除機能の実装 (フロントエンド)**

#### 4.1. 記事詳細ページ
- **作業ファイル:** `src/app/articles/[slug]/page.tsx` を新規作成する。
- **処理:**
    1.  URLの `slug` を元に、`/api/articles/[slug]` (GET) を呼び出して記事データを取得する。
    2.  取得した記事のタイトルと、Markdownで記述された本文を表示する。
    3.  記事所有者のみに表示される「編集」ボタンと「削除」ボタンを配置する。

#### 4.2. 記事編集ページ
- **作業ファイル:** `src/app/articles/[slug]/edit/page.tsx` を新規作成する。
- **処理:**
    1.  記事詳細ページと同様に、まず記事データを取得する。
    2.  新規作成ページと類似したフォームに、取得した既存データを初期値として設定する。
    3.  フォームが送信されたら、`/api/articles/[slug]` (PUT) に更新データを送信する。
    4.  更新が成功したら、記事詳細ページへリダイレクトする。

#### 4.3. 記事削除機能
- **処理:**
    1.  「削除」ボタンがクリックされたら、`daisyUI` のモーダルコンポーネントを使用して「本当に削除しますか？」という確認ダイアログを表示する。
    2.  ユーザーが確認ダイアログで「削除」を選択したら、`/api/articles/[slug]` (DELETE) を呼び出す。
    3.  削除が成功したら、記事一覧ページ (`/articles`) へリダイレクトする。

## 4. 検証方法

- [ ] 新規記事が作成でき、記事一覧に表示されること。
- [ ] 記事一覧から記事詳細ページに正しく遷移できること。
- [ ] 記事詳細ページから編集ページに遷移し、記事を更新できること。
- [ ] 記事を削除でき、一覧から消えること。
- [ ] 他人の記事を編集・削除しようとしても、ボタンが表示されない、またはAPIレベルで拒否されること。

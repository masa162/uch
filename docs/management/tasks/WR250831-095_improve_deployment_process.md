# 作業指示書: デプロイプロセスの改善

- **タスクID:** TSK-095
- **担当者:** Claude Code
- **期日:** N/A

## 1. 目的

データベースマイグレーションの実行に必要な `prisma` ディレクトリを、本番用のDockerイメージにビルド時から含めるように修正する。これにより、将来のデプロイ作業から `docker cp` を使った手動コピーの手順をなくし、プロセスを安全かつシンプルにする。

## 2. 作業手順

### ステップ1: `.dockerignore` の確認

まず、`.dockerignore` ファイルを確認し、`prisma` ディレクトリがビルド対象から意図せず除外されていないかを確認します。

**実行コマンド:**
```bash
cat .dockerignore
```

**確認内容:**
- ファイル内に `prisma` や `prisma/` といった記述がないことを確認してください。
- もし存在した場合は、その行を削除して保存してください。

### ステップ2: `Dockerfile` の修正

次に、`Dockerfile` を修正し、`prisma` ディレクトリをイメージ内に明示的にコピーする命令を追加します。

**対象ファイル:** `Dockerfile`

**改修方針:**
`COPY` 命令が並んでいる箇所（`COPY . .` の前などが望ましい）に、`prisma` ディレクトリをコピーする以下の1行を追加します。

```Dockerfile
# (他のCOPY命令など)

# Prismaスキーマとクライアントをコピー
COPY prisma ./prisma

# アプリケーションのコードをコピー
COPY . .

# (後続のビルドコマンドなど)
```

### ステップ3: 動作確認

修正が正しく行われたかを確認するため、ローカル環境で一度イメージをビルドし、新しいイメージ内に `prisma` ディレクトリが存在するかを確認します。

**実行コマンド:**
```bash
# 1. 新しいイメージをビルド
docker-compose build

# 2. ビルドされたイメージ内で ls コマンドを実行し、prismaディレクトリの存在を確認
# (uch-app は docker-compose.yml で定義されているサービス名です)
docker run --rm uch-app ls prisma
```

**期待される結果:**
`ls` コマンドの実行結果として、`schema.prisma` や `migrations` ディレクトリが表示されること。
エラーにならずに中身が表示されれば、修正は成功です。

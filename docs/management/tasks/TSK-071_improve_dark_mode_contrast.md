# TSK-071: [本番UI改善] モバイルのダークモードにおける文字コントラストを改善

## 概要
モバイル環境でのダークモード時の文字コントラスト問題を解決し、アクセシビリティを改善。

## 実施内容

### 1. 問題調査
- **現在の実装**: `data-theme="uchinokiroku"` 固定（ライトテーマ）
- **TailwindCSS設定**: `darkMode: 'media'` だが DaisyUI テーマが固定
- **問題**: モバイルでシステムダークモードが有効でも、ライトテーマが表示される

### 2. 根本原因特定
- **テーマ切り替え機能なし**: システム設定に応じたテーマ変更が実装されていない
- **DaisyUIダークテーマ未使用**: `dark` テーマは定義済みだが使われていない
- **モバイル固有問題**: ブラウザの自動変換により中途半端なコントラスト

### 3. 修正実装

#### A. ThemeProvider の実装
```typescript
// src/components/ThemeProvider.tsx
- MediaQuery API でシステム設定を監視
- `prefers-color-scheme: dark` に応じてテーマ切り替え
- 動的に `data-theme` 属性を更新
```

#### B. ダークテーマの改善
```typescript
// tailwind.config.ts - dark テーマ設定強化
"base-content": "#f3f4f6",  // テキストコントラスト改善
"base-100": "#111827",       // 背景色の調整
"base-200": "#1f2937",       // セカンダリ背景
```

#### C. CSS レベルでのコントラスト強化
```css
/* globals.css - モバイルダークモード対応 */
[data-theme="dark"] .text-gray-600 {
  color: #9ca3af !important;  // より明るいグレー
}
[data-theme="dark"] .text-gray-800 {
  color: #f3f4f6 !important;  // 高コントラスト白
}
```

### 4. 技術仕様

#### システム連携
- **MediaQuery監視**: `(prefers-color-scheme: dark)`
- **リアルタイム切り替え**: システム設定変更を即座に反映
- **初期化**: ページロード時にシステム設定を確認

#### テーマ管理
- **ライトモード**: `uchinokiroku` テーマ
- **ダークモード**: `dark` テーマ
- **動的切り替え**: `document.documentElement.setAttribute('data-theme', theme)`

#### コントラスト対応
- **WCAG準拠**: AA レベルのコントラスト比を確保
- **モバイル最適化**: 小さな画面での視認性向上
- **色覚配慮**: 色に依存しない情報伝達

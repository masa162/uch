# TSK-044: VPSファイアウォール設定の見直しと外部アクセス許可

## 1. タスク概要

TSK-043のデプロイ作業において、Dockerコンテナは正常に起動したものの、外部からアプリケーションへのHTTPアクセスがタイムアウトする問題が確認された。
本タスクでは、VPSのファイアウォール(UFW)の設定を見直し、指定ポートへの外部からのアクセスを許可することで、アプリケーションを公開状態にすることを目的とする。

## 2. 背景

- **関連タスク**: TSK-043_vps_initial_deployment.md
- **問題**: `curl http://<VPSのIPアドレス>:3000` がタイムアウトする。
- **仮説**: ConoHa VPSの標準設定、あるいはOS(Ubuntu)のファイアウォール(UFW)が、ポート3000への外部からの着信トラフィックをブロックしている可能性が高い。

## 3. 作業内容

1.  **UFWのステータス確認**:
    -   VPSにSSHで接続し、以下のコマンドでUFWの現在のルールを確認する。
        ```bash
        sudo ufw status verbose
        ```
2.  **ポート3000の許可設定**:
    -   アプリケーションが使用するポート3000へのTCP接続を許可するルールを追加する。
        ```bash
        sudo ufw allow 3000/tcp
        ```
    -   より限定的に、特定のIPアドレスからのみ許可する場合は以下のように指定する。
        ```bash
        # 例: 特定のIPアドレスからのみ許可
        # sudo ufw allow from <your_ip_address> to any port 3000 proto tcp
        ```
        今回は、一般公開を目的とするため、全IPからのアクセスを許可する。
3.  **設定の反映と確認**:
    -   追加したルールを有効化する。
        ```bash
        sudo ufw reload
        ```
    -   再度ステータスを確認し、ポート3000の許可ルールが追加されていることを確認する。
        ```bash
        sudo ufw status
        ```
4.  **外部からの接続テスト**:
    -   ローカルマシンから`curl`コマンドまたはブラウザで、VPSのIPアドレスとポート3000にアクセスし、正常にレスポンスが返ってくることを確認する。
        ```bash
        curl -I http://160.251.136.92:3000
        ```

## 4. 完了条件

-   ローカル環境からVPS上のアプリケーション（`http://160.251.136.92:3000`）にアクセスし、HTTPステータスコード200のレスポンスが返ってくること。
-   ブラウザでアクセスし、アプリケーションの初期画面が表示されること。
